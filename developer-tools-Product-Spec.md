# developer-tools - Product Spec

## 1. 产品概述

### 1.1 产品定位
一个集合了加密解密、编码转换、JSON处理、Web开发、提示词管理等 50+ 种开发者工具的在线工具网站，类似于 it-tools.tech 和 aishort.top 的结合体。

### 1.2 目标用户
- **主要用户**：前端/后端开发者、运维工程师、产品经理等技术人员
- **使用场景**：
  1. 开发过程中需要快速加密解密文本、生成 UUID、处理 JSON 数据
  2. 需要将 YAML 转换为 JSON、将 Markdown 转换为 HTML
  3. 查看、搜索、复制常用的 AI 提示词模板
  4. 查看、搜索、复制常用的 AI 技能（Skills）定义
  5. 查看、搜索、复制常用的开发规范（Rules）

### 1.3 核心价值
开发者日常工作中需要用到各种小工具，但通常需要在多个网站之间切换。developer-tools 将这些工具整合在一起，提供一站式的服务，同时增加了 AI 提示词优化功能，提升工作效率。

## 2. 功能需求

### 2.1 核心功能列表

#### 2.1.1 加密解密工具类（Crypto）
- **Token Generator** - 生成随机 Token
- **Hash Text** - 文本哈希（MD5、SHA1、SHA256 等）
- **Bcrypt** - Bcrypt 加密/验证
- **UUIDs Generator** - 生成 UUID
- **ULID Generator** - 生成 ULID
- **Encrypt / Decrypt Text** - 对称加密解密文本
- **BIP39 Passphrase Generator** - 生成 BIP39 助记词
- **Hmac Generator** - 生成 HMAC
- **RSA Key Pair Generator** - 生成 RSA 密钥对
- **Password Strength Analyser** - 分析密码强度
- **PDF Signature Checker** - 检查 PDF 签名

#### 2.1.2 转换工具类（Converter）
- **Date-time Converter** - 日期时间转换
- **Integer Base Converter** - 整数进制转换
- **Roman Numeral Converter** - 罗马数字转换
- **Base64 String Encoder/Decoder** - Base64 字符串编码解码
- **Base64 File Converter** - Base64 文件转换
- **Color Converter** - 颜色格式转换（HEX、RGB、HSL 等）
- **Case Converter** - 大小写转换
- **Text to NATO Alphabet** - 文本转 NATO 字母表
- **Text to ASCII Binary** - 文本转 ASCII 二进制
- **Text to Unicode** - 文本转 Unicode
- **YAML to JSON** - YAML 转 JSON
- **YAML to TOML** - YAML 转 TOML
- **JSON to YAML** - JSON 转 YAML
- **JSON to TOML** - JSON 转 TOML
- **List Converter** - 列表格式转换
- **TOML to JSON** - TOML 转 JSON
- **TOML to YAML** - TOML 转 YAML
- **XML to JSON** - XML 转 JSON
- **JSON to XML** - JSON 转 XML
- **Markdown to HTML** - Markdown 转 HTML

#### 2.1.3 Web 开发工具类（Web）
- **Encode/Decode URL** - URL 编码解码
- **Escape HTML Entities** - HTML 实体转义
- **URL Parser** - URL 解析
- **Device Information** - 设备信息检测
- **Basic Auth Generator** - Basic Auth 生成器
- **Open Graph Meta Generator** - Open Graph 元标签生成器
- **OTP Code Generator** - OTP 验证码生成器
- **MIME Types** - MIME 类型查询
- **JWT Parser** - JWT 解析器
- **Keycode Info** - 键码信息查询
- **Slugify String** - Slug 化字符串
- **HTML WYSIWYG Editor** - HTML 所见即所得编辑器
- **User Agent Parser** - User Agent 解析
- **JSON Viewer** - JSON 查看器/格式化器

#### 2.1.4 Cron 工具类
- **Cron Expression Generator** - Cron 表达式生成器
- **Cron Expression Parser** - Cron 表达式解析

#### 2.1.5 提示词管理工具类（Prompts）
- **提示词浏览** - 浏览内置提示词模板列表
- **提示词搜索** - 按关键词搜索提示词
- **提示词复制** - 一键复制提示词内容
- **提示词分享** - 生成提示词分享链接
- **提示词详情** - 查看提示词详细内容和使用说明

#### 2.1.6 Skills 管理工具类（Skills）
- **Skills 浏览** - 浏览内置 Skills 列表
- **Skills 搜索** - 按关键词搜索 Skills
- **Skills 复制** - 一键复制 Skills 内容
- **Skills 分享** - 生成 Skills 分享链接
- **Skills 详情** - 查看 Skills 详细内容

#### 2.1.7 Rules 管理工具类（Rules）
- **Rules 浏览** - 浏览开发规范列表
- **Rules 搜索** - 按关键词搜索 Rules
- **Rules 复制** - 一键复制 Rules 内容
- **Rules 分享** - 生成 Rules 分享链接
- **Rules 详情** - 查看 Rules 详细内容

### 2.2 AI 增强功能

#### AI 增强点1：提示词优化
- **原始功能**：用户编写提示词
- **AI 增强方式**：AI 一键优化提示词（改写、补充、完善）
- **使用场景**：用户对自己的提示词效果不满意时，点击"AI 优化"按钮

#### AI 增强点2：智能推荐工具
- **原始功能**：用户手动搜索工具
- **AI 增强方式**：根据用户描述的需求，AI 推荐合适的工具
- **使用场景**：用户不确定该使用哪个工具时

## 3. 用户流程

### 3.1 主要用户流程

**流程名称：使用开发者工具流程**

1. **步骤1：首页/工具列表页**
   - 用户操作：打开网站，看到工具分类列表
   - 系统响应：展示左侧分类导航 + 右侧工具卡片
   - 界面状态：显示"编程工具"、"加密工具"、"转换工具"、"Web工具"、"提示词"、"Skills"、"Rules"等分类

2. **步骤2：搜索工具**
   - 用户操作：在顶部搜索框输入关键词
   - 系统响应：实时过滤显示匹配的工具
   - 界面状态：搜索结果高亮显示

3. **步骤3：进入工具页面**
   - 用户操作：点击工具卡片，进入具体工具页面
   - 系统响应：跳转到工具详情页
   - 界面状态：显示工具名称、描述、输入区域、输出区域

4. **步骤4：使用工具**
   - 用户操作：在输入区域输入内容，点击"处理"按钮
   - 系统响应：处理完成后在输出区域显示结果
   - 界面状态：显示处理结果，提供复制按钮

**流程名称：提示词/Skills/Rules 查看流程**

1. **步骤1：进入提示词/Skills/Rules 页面**
   - 用户操作：从左侧导航点击"提示词"或"Skills"或"Rules"
   - 系统响应：展示列表页，显示内置的提示词/Skills/Rules
   - 界面状态：卡片式列表展示

2. **步骤2：浏览和搜索**
   - 用户操作：浏览列表或使用搜索功能
   - 系统响应：显示匹配的条目
   - 界面状态：搜索结果实时更新

3. **步骤3：查看详情**
   - 用户操作：点击任意条目查看详情
   - 系统响应：显示详细内容弹窗或新页面
   - 界面状态：显示标题、描述、完整内容

4. **步骤4：复制或分享**
   - 用户操作：点击"复制"或"分享"按钮
   - 系统响应：复制到剪贴板或生成分享链接/二维码
   - 界面状态：显示复制成功或分享链接

### 3.2 异常流程

**异常情况1：工具处理失败**
- **触发条件**：输入格式不正确或处理出错
- **处理方式**：显示错误提示信息，说明错误原因

**异常情况2：分享链接无效**
- **触发条件**：访问不存在的分享链接
- **处理方式**：显示"链接无效"提示，返回首页

## 4. UI/UX 设计

### 4.1 整体布局

**布局类型**：两栏布局（左侧固定导航 + 右侧自适应内容）

**布局结构**：
```
┌──────────────────────────────────────────────────┐
│                    Header                         │
│              (顶部搜索框 + 标题)                  │
├─────────────┬────────────────────────────────────┤
│             │                                    │
│   Sidebar   │         Main Content              │
│   (左侧导航) │          (右侧内容)                │
│             │                                    │
│  - 工具分类  │   工具卡片列表 / 工具详情页面     │
│  - 提示词    │                                    │
│  - Skills   │                                    │
│  - Rules    │                                    │
│             │                                    │
└─────────────┴────────────────────────────────────┘
```

**各区域说明**：
- **Header**：高度 64px，包含网站 Logo、标题、全局搜索框
- **Sidebar**：宽度 240px（可折叠），包含工具分类导航、提示词、Skills、Rules 入口
- **Main Content**：自适应宽度，最小 600px，内容区根据页面动态变化

### 4.2 页面详细设计

#### 页面1：首页（工具列表页）

**布局**：
- 左侧：固定分类导航（240px）
- 右侧：工具卡片网格（3-4 列自适应）

**左侧导航**：
- 收起/展开按钮
- 分类列表（Crypto、Converter、Web、Cron 等）
- 提示词入口
- Skills 入口
- Rules 入口

**右侧内容**：
- 搜索结果过滤时：显示匹配的工具卡片
- 全部显示时：按分类分组显示工具卡片

**工具卡片**：
- 尺寸：宽度自适应，最小 280px
- 内容：图标、工具名称、简短描述、状态标签
- 交互：hover 时显示阴影，点击跳转详情页

#### 页面2：工具详情页

**布局**：
- 左侧：工具名称 + 描述
- 中间：输入区域（占 50%）
- 右侧：输出区域（占 50%）

**输入区域**：
- 文本输入框：铺满容器，高度 200px，可调整
- 选项配置：根据工具不同显示不同选项
- 操作按钮："处理"、"清空"、"复制"

**输出区域**：
- 结果展示框：只读，高度与输入区一致
- 操作按钮："复制"、"下载"（如适用）

**交互细节**：
- 点击"处理"后显示加载状态
- 处理完成后自动复制结果（可选）
- 支持快捷键（Ctrl+Enter 提交）

#### 页面3：提示词/Skills/Rules 列表页

**布局**：
- 搜索框在顶部
- 卡片网格展示（2-3 列）

**卡片内容**：
- 标题
- 分类标签
- 简短描述
- 操作按钮：查看、复制、分享

#### 页面4：提示词/Skills/Rules 详情页

**布局**：
- 顶部：标题 + 分类标签
- 中间：完整内容展示（代码块样式）
- 底部：操作按钮（复制、分享）

### 4.3 控件规范

**输入框**：
- 宽度：铺满容器（flex: 1）
- 高度：200px（可调整）
- 样式：边框 1px solid #d9d9d9，圆角 6px，内边距 12px

**按钮**：
- 主要按钮：背景 #1677ff，白色文字，右对齐
- 次要按钮：边框 #d9d9d9，黑色文字
- 位置：输入区域底部右侧

**搜索框**：
- 宽度：300px（Header 中）/ 铺满（列表页顶部）
- 高度：36px
- 占位符："搜索工具..."
- 图标：搜索图标在左侧

## 5. Google AI Studio 能力配置

### 5.1 使用的 AI 能力列表

| AI 能力 | 用途 | 使用场景 |
|---------|------|----------|
| gemini-2.0-flash | 提示词优化 | 用户点击"AI 优化"按钮时，对提示词进行改写和优化 |
| gemini-2.0-flash | 智能工具推荐 | 用户在搜索框输入需求描述时，AI 推荐合适的工具 |

### 5.2 能力配置说明

**能力1：gemini-2.0-flash（提示词优化）**
- **配置参数**：
  - temperature: 0.7
  - maxTokens: 2048
- **调用时机**：用户点击"AI 优化"按钮时
- **输入格式**：用户当前的提示词内容 + 优化指令
- **输出格式**：优化后的提示词文本

**能力2：gemini-2.0-flash（智能工具推荐）**
- **配置参数**：
  - temperature: 0.3
  - maxTokens: 512
- **调用时机**：用户搜索时输入非工具名的自然语言描述
- **输入格式**：用户的需求描述
- **输出格式**：推荐的工具名称列表

## 6. 技术需求

### 6.1 技术要求
- Next.js 14+ (App Router)
- React 18+
- TypeScript
- UnoCSS (替代 Tailwind CSS)
- Biome.js (替代 ESLint + Prettier + Stylelint)
- 纯前端实现，无需后端
- 支持 PWA（可选）

### 6.2 性能要求
- 首屏加载时间 < 2s
- 工具处理响应时间 < 1s（纯前端处理）
- AI 功能响应时间 < 5s

### 6.3 数据要求
- 工具配置：本地 JSON 文件存储
- 提示词/Skills/Rules 数据：本地 JSON 文件存储
- 分享链接：通过 URL 参数传递数据（无需后端存储）
